## 各模块详细说明

### 模块 1：头部卡片 — 身份区

| 字段 | 类型 | 说明 |
|------|------|------|
| `uid` | string | 客户UID，8位数字，页面主标识，支持点击复制 |
| `status` | enum | 流失(红) / 增长(绿) / 稳定(灰)，圆角 pill 样式 |
| `statusBadge.bg` | string | 状态标签背景色（rgba） |
| `statusBadge.text` | string | 状态标签文字色 |
| `from` | enum | 来源标签，URL参数传入：event=事件、clients=列表、search=搜索 |
| `isFav` | boolean | 收藏状态，localStorage 持久化，key=`fav_{uid}` |

**数据来源**：`uid` 和 `status` 从后端获取；`from` 来自前端路由参数；`isFav` 为前端本地存储。

---

### 模块 2：头部卡片 — 资沉指标

| 字段 | 类型 | 说明 |
|------|------|------|
| `thisWeek` | number | 本周资沉 = `weeks[8]`，页面最大数字（28px 加粗） |
| `thisWeekDisplay` | string | 格式化展示：>= 1亿显示"X.X亿"，>= 1万显示"X万" |
| `fMean` | number | 9周均值，辅助参考 |
| `depositDisplay` | string | fMean 格式化，"8周均 $XX万" |
| `deltaPct` | number | 偏离百分比，带方向箭头和颜色 |
| `deltaAbs` | number | 偏离绝对值 |

**数据来源**：`weeks[]` 从资产系统获取周快照；其余为前端派生计算。

---

### 模块 3：头部卡片 — 备忘

| 字段 | 类型 | 说明 |
|------|------|------|
| `memo` | string | BD 一句话备忘，输入即保存 |

**数据来源**：`localStorage('memo_' + uid)`，纯前端本地。真实产品应考虑同步到后端。

### 模块 4：事件 — 当前事件列表

每条事件的完整字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 事件唯一ID `evt-{uid}-{seq}` |
| `type` | string | 事件类型：风控触发 / 超大额出金 / 爆仓 |
| `sub_type` | string | 子类型，风控=提现限制/交易限制/账户冻结/KYC异常，爆仓=合约爆仓/杠杆爆仓 |
| `urgency` | enum | 紧急度 critical/warning/info |
| `color` | string | 紧急度颜色 |
| `amount` | number | 涉及金额（负值=出金/亏损，0=风控无金额） |
| `amount_display` | string | 金额格式化展示 |
| `time_ago` | string | 相对时间描述 |
| `age_hours` | number | 事件年龄（小时），用于过期判断 |
| `task_state` | enum | 待处理 / 跟进中 / 已过期 |
| `expires_in` | string/null | 过期窗口 |
| `client_uid` | string | 所属客户UID |
| `client_name` | string | 所属客户名 |
| `risk_type` | string | 风控子类型（仅风控事件） |
| `action` | string | 建议动作 |
| `script` | string/null | 话术模板 |

**已处理事件过滤**：页面通过 `sessionStorage('processed_events')` 中的 ID 数组过滤已处理事件。

**数据来源**：真实产品需要风控系统（风控触发）、充提系统（超大额出金）、交易系统（爆仓）分别推送事件。

### 模块 5：事件 — 跟进记录

| 字段 | 类型 | 说明 |
|------|------|------|
| `date` | string | 跟进日期 "MM-DD HH:mm" |
| `bd` | string | 执行人，人工="BD 李明远"，系统="系统" |
| `text` | string | 跟进内容文本 |

**展示规则**：时间线样式，人工=蓝色圆点，系统=灰色圆点，最新在最上面。

**数据来源**：MVP 为硬编码假数据（按 status 区分模板）。真实产品需要跟进记录系统。

### 模块 6：资产 — 资金分布

6 个业务模块各自的余额和占比：

| 字段 | 类型 | 说明 |
|------|------|------|
| `depositDist.futures` / `futuresDisplay` | number / string | 双仓合约余额 |
| `depositDist.leverage` / `leverageDisplay` | number / string | 超级杠杆余额 |
| `depositDist.savings` / `savingsDisplay` | number / string | 理财余额 |
| `depositDist.card` / `cardDisplay` | number / string | 金融卡余额 |
| `depositDist.cloud` / `cloudDisplay` | number / string | 云算力余额 |
| `depositDist.mini` / `miniDisplay` | number / string | Mini合约余额 |

**展示规则**：按金额降序排列，前两名进度条金色，其余蓝色。顶部显示总计（= fMean）。

**数据来源**：真实产品需要分别查询 6 个业务系统的账户余额。MVP 通过 `_deposit_distribution()` 按随机权重分配 fMean。

### 模块 7：资产 — 资沉流动（双向柱状图）

| 字段 | 类型 | 说明 |
|------|------|------|
| `flowWeeks[0..7]` | object[] | 8周的流入/流出数据（W2-本周） |
| `flowWeeks[i].inflow` | number | 第i周流入金额 |
| `flowWeeks[i].outflow` | number | 第i周流出金额 |
| `avgInflowPerWeek` | number | 前7周平均每周流入 |
| `avgOutflowPerWeek` | number | 前7周平均每周流出 |
| `avgNetPerWeek` | number | 前7周平均每周净流量 |

**展示规则**：上绿（流入）下红（流出）双向柱状图，X轴 W2-W8+本周，默认展开。

**数据来源**：
- **MVP**：根据 `weeks[]` 相邻差值模拟，见算法 7
- **真实产品**：从充提系统获取每周充值/提现记录汇总

## 算法详细说明

### 算法 1：客户状态判定

```
输入：weeks[0..8]（9周资沉序列）

1. 线性回归：对 t=[1..9] vs weeks 做最小二乘回归
   slope = Σ((t-t̄)(y-ȳ)) / Σ((t-t̄)²)
   R² = 1 - SS_res / SS_tot

2. 趋势率：trend_rate = slope / fMean

3. 平台调整：adjusted_rate = trend_rate - platform_trend_rate

4. 置信度：conf = 0.5 + 0.5 * R²

5. 评分（log 压缩）：
   t_f = 80分位资沉值
   log_f = ln(1 + fMean / t_f)
   churn_score = log_f * max(0, -adjusted_rate - 0.02) * conf
   growth_score = log_f * max(0, adjusted_rate - 0.02) * conf

6. 判定：
   churn_score > 0 → 流失
   growth_score > 0 → 增长
   否则 → 稳定
```

### 算法 2：资沉8周均值

```
fMean = (w1 + w2 + ... + w9) / 9
```

注：实际是 9 个周数据点的平均值，文案展示为"8周均"。

### 算法 3：Delta 百分比

```
thisWeek = weeks[8]  // 第9周（最新一周）
deltaPct = (thisWeek - fMean) / fMean × 100%
```

### 算法 4：Delta 绝对值

```
deltaAbs = thisWeek - fMean
```

### 算法 5：资金分布（MVP 模拟）

```python
# 仅客户参与的业务线分配非零权重
for module in [futures, leverage, savings, card, cloud, mini]:
    if client.has_{module}:
        weight[module] = random(0.01, 1.0)
    else:
        weight[module] = 0

# 按权重比例分配 fMean
module_balance = fMean * weight[module] / sum(weights)
```

**真实产品**：直接查询各业务系统的账户余额，不需要按权重分配。

### 算法 6：9周资沉序列生成（MVP 模拟）

```python
base = exp(gauss(15.4, 1.8))  # 中位数约 500万 USDT

for t in range(9):
    if trend_type == "churn":
        drift = -random(0.01, 0.06) * (t+1)
    elif trend_type == "growth":
        drift = random(0.01, 0.05) * (t+1)
    else:
        drift = random(-0.02, 0.02) * (t+1)
    noise = gauss(0, 0.05)
    weeks[t] = max(base * (1 + drift + noise), 1000)
```

**真实产品**：从资产系统获取每周末资沉快照。

### 算法 7：资沉流动（流入/流出模拟，MVP）

```javascript
// 基于相邻周资沉差值模拟流入/流出
for (i = 1; i < weeks.length; i++) {
    delta = weeks[i] - weeks[i-1]
    r = seededRandom(uid + i)  // 确定性随机

    if (delta > 0) {
        inflow = delta * (1.2 + r * 0.6)   // 流入 > 净增
        outflow = inflow - delta             // 差额 = 同期流出
    } else if (delta < 0) {
        outflow = |delta| * (1.2 + r * 0.6) // 流出 > 净减
        inflow = outflow - |delta|            // 差额 = 同期流入
    } else {
        inflow = fMean * (0.01 + r * 0.04)   // 小幅双向流动
        outflow = inflow * (0.95 + r2 * 0.1)
    }
}
```

**真实产品**：从充提系统获取每周充值记录总和（= 流入）和提现记录总和（= 流出），不需要模拟。

### 算法 8：风控触发事件生成

```
触发条件：status == '流失' AND random() < 0.3（30%概率）
金额：0（风控事件无金额）
过期窗口：不过期（直到解除）
risk_type 随机选择：提现限制 / 交易限制 / 账户冻结 / KYC异常
```

**真实产品**：由风控系统实时推送，触发条件由风控规则引擎决定。

### 算法 9：超大额出金事件生成

```
触发条件：status == '流失' AND random() < 0.4（40%概率）
金额：fMean * random(0.3, 0.6)，即资沉的 30%-60%
过期窗口：48小时
过期判定：age_hours > 48 → task_state = '已过期'
```

**真实产品**：由充提系统在出金审核时触发，条件为单笔出金 >= 资沉 * 30%。

### 算法 10：爆仓事件生成

```
触发条件：(has_futures OR has_leverage) AND random() < 0.2（20%概率）
金额：fMean * random(0.05, 0.2)，即资沉的 5%-20%
子类型：从客户参与的业务线中选择（合约爆仓/杠杆爆仓）
过期窗口：72小时
过期判定：age_hours > 72 → task_state = '已过期'
```

**真实产品**：由交易系统在强制平仓时推送。

### 算法 11：事件过期规则

| 事件类型 | 过期窗口 | 逻辑 |
|---------|---------|------|
| 风控触发 | 不过期 | `expires_in = null`，直到手动解除 |
| 超大额出金 | 48小时 | `age_hours > 48` → 自动标记已过期 |
| 爆仓 | 72小时 | `age_hours > 72` → 自动标记已过期 |

处理闭环：BD 点击"标记已处理" → 事件ID 写入 `sessionStorage('processed_events')` → 首页和详情页过滤。

### 算法 12：综合净贡献公式

```
净贡献 = 合约收入 + 杠杆收入 + Mini合约收入 + 金融卡收入 + 云算力收入
        - |理财利息支出|
        - |返佣支出|
```

各模块收入明细：
- **合约/杠杆**：对冲单=手续费，非对冲单=手续费+用户亏损（用户盈利则减去）
- **Mini合约**：开仓手续费 + 平仓手续费 + 资金费率 - 用户盈亏
- **金融卡**：开卡费 + 消费手续费
- **云算力**：算力费用 + 电费
- **理财利息**：已发放利息总额（平台支出，取负值）
- **返佣**：已结算返佣总额（平台支出，取负值）

MVP 模拟生成逻辑：

```python
base = fMean * random(0.001, 0.02)  # 净贡献基数
futures = base * random(0.3, 0.6)
leverage = base * random(0.1, 0.3)
mini = base * random(0.05, 0.15)
card = base * random(0.02, 0.1)
cloud = base * random(0.01, 0.08)
savings_interest = -base * random(0.01, 0.05)  # 负值
rebate = -base * random(0.005, 0.03)           # 负值
total = sum(以上)
wow = random(-0.3, 0.4)  # 周环比随机
```
