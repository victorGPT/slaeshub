## 核心算法说明

### 1. 综合净贡献

平台从该客户身上赚了多少净利润。

```
净利润 = 双仓合约 + 超级杠杆 + Mini合约 + 金融卡 + 云算力 - 理财利息 - 返佣
```

**5 个收入模块**：
- 双仓合约：手续费 + 非对冲订单的用户亏损
- 超级杠杆：手续费 + 非对冲订单的用户亏损
- Mini合约：手续费 + 资金费率 - 用户盈亏
- 金融卡：开卡费 + 消费手续费
- 云算力：算力费用 + 电费

**2 个成本模块**：
- 理财利息：平台付给客户的利息（纯支出）
- 返佣：平台付给渠道代理的佣金（纯支出）

### 2. 事件触发条件

| 事件类型 | 触发条件 | 过期窗口 |
|---------|---------|---------|
| 风控触发 | 风控系统标记账户异常（冻结/限制/KYC） | 不过期 |
| 超大额出金 | 单笔出金 ≥ 客户资沉的 30% | 48 小时 |
| 爆仓 | 合约或杠杆仓位被强制平仓 | 72 小时 |

### 3. 资沉流动的流入/流出

- **流入** = 客户从外部向平台充值的金额
- **流出** = 客户从平台向外部提现的金额
- 与资沉变化不同：资沉还受交易盈亏、币价波动等影响，流入/流出只计算真实的资金进出
