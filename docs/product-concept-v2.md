# Sales Hub 产品概念 v2 — 以资金沉淀为北极星

## 1. 产品定位

让 BD（商务员）围绕客户资金沉淀，高效完成三件事：
- 看清盘子（谁最重要）
- 止损流失（谁在掉）
- 放大增长（谁在涨）

## 2. 北极星指标

```text
F = 客户在平台的总资金沉淀（折 USDT，每日 UTC 00:00 快照）
```

资金沉淀不是辅助指标，而是前台主入口。资沉收益 >> 手续费收益，所以 F 本身就是价值。

## 3. 前台结构

一个主列表 + 两个快捷筛选 Tab：

| Tab | 默认排序 | 入池条件 |
|-----|---------|---------|
| 资沉总量（默认） | F_base 降序 | 全量客户 |
| 资沉流失 | ChurnPriority 降序 | F_base >= T_F 且 AdjustedTrend_F <= -δ 且 AbsDrop >= T_drop |
| 资沉增长 | GrowthPriority 降序 | F_base >= T_F 且 AdjustedTrend_F >= δ 且 AbsGrowth >= T_growth |

## 4. 核心数据口径

### 4.1 资沉估值
- 统一 USDT 计价
- V_t(u) = Σ(q_i,t × p_i,t)，按持仓金额加权求和

### 4.2 时间窗
- F_base：近 30 天日均资沉
- F_recent：近 7 天日均资沉

### 4.3 趋势（含市场修正）
```text
Trend_F = (F_recent - F_base) / max(F_base, ε)
PlatformTrend = 平台整体资沉变化
AdjustedTrend_F = Trend_F - PlatformTrend
```

只有跑输/跑赢平台整体的客户才算真实流失/增长。

### 4.4 双门槛入池
流失和增长视图同时要求：
- 相对变化 >= δ（建议 10%）
- 绝对金额 >= T_drop / T_growth（建议 50,000 USDT）

## 5. 排序公式

```text
DepositRank = F_base
ChurnPriority = F_base × |AdjustedTrend_F|
GrowthPriority = F_base × AdjustedTrend_F
```

## 6. 客户卡片（四项信息）

1. 当前资沉总量
2. 近期状态（增长 / 稳定 / 流失）
3. 原因标签
4. 建议动作

## 7. 动作约束

- 冷却：同类动作 7 天内不重复
- 互斥：同一时点只有一个主动作
- 优先级：先止损，再增长
- 留痕：每次动作记录前状态 + 事件 + 7/30 天后结果

## 8. 动作白名单

1. 销售回访
2. 续存提醒 / 权益说明
3. 资沉补贴券
4. U 卡 / 金融卡积分活动
5. 场景绑定说明

## 9. 视图冲突规则

- 流失优先于增长（先止损再增长）
- 同一客户只在一个视图中出现
- 不满足流失/增长条件的留在总量视图

## 10. MVP 不做清单

- 不做高价值 / 高潜力互斥分类
- 不做复杂客户画像体系
- 不做多业务多路径推荐
- 不做 Top-3 推荐动作
- 不做登录活跃度、可干预系数进入主排序
- 不做迁移机制
- 不做复杂 AI 归因
- 不做依赖人工标注的数据字段
