## §1 术语定义

| 术语 | 定义 |
|------|------|
| **BD** | 使用本系统的业务拓展人员 |
| **客户** | BD 名下管理的交易用户，以 UID 唯一标识 |
| **滚动7日** | 以当日为终点、向前回溯 7 个自然日的时间窗口；**不按日历周（周一至周日）对齐**，每天计算时窗口自动前移。本系统所有「7日」指标均采用此定义，非自然周 |
| **资沉** | 客户过去滚动7日的持仓资产均值（单位 USDT） |
| **7天贡献** | 交易所过去滚动7日内从该客户获得的盈利（对应字段 `netContribution.total`） |
| **滚动8周均值** | 过去 8 个滚动7日窗口的指标周平均值，用于计算环比变化百分比 |
| **变化%** | 当前值与滚动8周均值的差值比率，正值为绿色、负值为红色 |
| **待办** | BD 对事件执行「加入待办」操作后、且未关闭的事件集合（见 PRD_事件中心_MVP.md §1） |
| **事件** | 系统自动触发或 BD 手动创建的工作单元（见 PRD_事件中心_MVP.md §1） |
| **创建时间** | 事件被创建的时间，待办列表按此字段排序 |
| **跑马灯** | 事件列表超过 3 条时自动垂直滚动的展示方式 |
| **用户排名** | 按资沉降序排列的客户榜单，最多展示 Top 10 |

## §2 模块概述

### §2.1 定位

首页是 BD 进入系统后的默认落点，提供当日工作的全局视图：待处理的待办事项、最新发生的客户事件、以及资产规模排名 Top 10 的客户概览。

### §2.2 目标平台

移动端 Web App（iOS/Android 浏览器 PWA，设计宽度 393px）。

### §2.3 前置条件

BD 已登录系统，系统已完成客户数据的初始化加载。

### §2.4 MVP 范围

**做**
- 顶部导航栏（Logo + 页面标题 + 个人中心入口）
- 搜索栏（客户名 / UID 快速检索）
- 事件模块（最新事件跑马灯预览）
- 待办模块（待办事件摘要列表）
- 用户排名模块（资沉 Top 10 表格）
- 底部 Tab 导航栏

**不做**
- 首页内容个性化配置（模块顺序/显示隐藏）
- 多维度排名切换（MVP 仅支持资沉降序）
- 未读消息红点 / Push 推送
- 下拉刷新（排名刷新时间随页面加载自动更新）
- 登录 / 注册页面（待单独建设，当前原型以退出登录占位）

### §3.1 顶部导航栏

#### 功能说明

固定在页面顶部，提供品牌标识、当前页面标题和个人中心入口。

#### 触发方式

进入首页时始终展示，不可折叠。

#### 页面结构

| 位置 | 元素 | 说明 |
|------|------|------|
| 左侧 | 品牌缩写「SH」| 金色，字体加粗 |
| 中间 | 页面标题「首页」| 主文案 |
| 右侧 | 个人中心图标 | 点击跳转至个人中心页 |

#### 字段说明

无动态字段，全部为静态展示内容。

#### 操作结果

- 点击右侧账户图标 → 跳转至账户页（profile.html）

#### 账户页现状说明

> ⚠️ **登录 / 注册页面尚未建设**，原型中账户页（profile.html）目前仅包含「退出登录」入口作为占位。正式上线前需补充：
> - 未登录状态：展示登录 / 注册入口
> - 已登录状态：展示账户信息 + 退出登录

#### 边界条件

无特殊边界，该栏始终可见。

### §3.2 搜索栏

#### 功能说明

BD 可在首页通过客户姓名或 UID 快速检索客户，检索结果在输入框下方以下拉列表展示，点击后跳转至对应客户详情页。

#### 触发方式

点击搜索栏输入框，键入字符后实时触发检索。

#### 页面结构

| 元素 | 状态 | 说明 |
|------|------|------|
| 搜索输入框 | 默认 | 展示占位文案「搜索客户名字或 UID...」 |
| 搜索输入框 | 输入中 | 展示已输入内容，同时右侧出现清除按钮 |
| 清除按钮（×）| 有内容时显示 | 点击后清空输入框并关闭下拉列表 |
| 下拉结果列表 | 有匹配结果 | 展示至多 20 条匹配客户 |
| 下拉结果列表 | 无匹配结果 | 展示「未找到匹配客户」提示 |
| 下拉结果列表 | 输入为空 | 不显示 |

#### 字段说明（下拉结果每行）

| 字段 | 显示条件 | 说明 |
|------|---------|------|
| UID | 始终显示 | 客户唯一标识，主要文案 |
| 备注姓名 | 仅当操作者在客户档案中已添加备注姓名时显示 | 次要文案，灰色小字；未设置则不占位 |

> 搜索结果行**不展示**持仓资产、状态标签等其他指标。

#### 操作结果

- 输入字符 → 实时过滤，下拉列表更新
- 点击结果行 → 跳转至对应客户详情页
- 点击清除按钮 / 按 ESC 键 → 清空输入，关闭下拉列表，焦点留在输入框
- 搜索词命中规则：姓名或 UID 包含输入字符（大小写不敏感），结果行仅展示 UID

#### 边界条件

| 场景 | 处理 |
|------|------|
| 输入框为空 | 下拉列表不出现 |
| 匹配结果超过 20 条 | 只展示前 20 条，不提示截断 |
| 无任何匹配 | 展示「未找到匹配客户」，不展示空列表 |
| 已跳转到结果页后返回 | 搜索框恢复为空，下拉列表关闭 |

### §3.3 事件模块

#### 功能说明

在首页以预览列表形式展示最新事件，最多同时可见 3 行；事件数量超过 3 条时启用自动垂直滚动（跑马灯），让 BD 无需离开首页即可感知最近动态。提供跳转全部事件的入口。

#### 触发方式

进入首页时自动渲染。跑马灯在事件数量 > 3 时自动启动；鼠标悬停（或触摸长按）时暂停。

#### 页面结构

| 元素 | 说明 |
|------|------|
| 模块标题行 | 🔔 图标 + 「事件」文字 + 事件总数角标 |
| 事件列表区域 | 固定高度，最多同时展示 3 行；超过 3 条时滚动 |
| 单条事件行 | 事件类型角标 + 事件标题 + 客户信息 |
| 「查看全部事件」入口 | 固定在列表底部，始终显示，点击跳转事件中心页 |
| 空状态提示 | 无事件时展示「暂无事件」 |

#### 字段说明（每条事件行）

| 字段 | 说明 |
|------|------|
| 事件类型角标 | 文字标签 + 颜色编码（见下表）；无法识别类型时显示「事件」（灰色） |
| 事件标题 | 事件简要描述，超长截断 |
| 客户信息 | 客户姓名（或 UID）+ 昵称（如有则以「·」分隔拼接） |

#### 事件类型颜色规则

| 事件类型 | 角标颜色 |
|---------|---------|
| 大额充值 | 绿色 |
| 大额盈利 | 绿色 |
| 大额提现 | 红色 |
| 大额亏损 | 红色 |
| 其他类型 | 灰色 |

#### 滚动规则

| 条件 | 行为 |
|------|------|
| 事件数 ≤ 3 | 静态展示，无动画 |
| 事件数 > 3 | 启用垂直滚动，列表无缝循环；滚动速度 = 事件数 × 2.5 秒一轮 |
| 鼠标悬停 / 触摸长按 | 滚动暂停 |
| 鼠标移开 / 触摸抬起 | 滚动继续 |

#### 展示顺序

按事件创建时间降序排列，最新事件排最前。

#### 操作结果

- 点击任意事件行 → 跳转至对应事件详情页（follow-up-thread）
- 点击「查看全部事件」→ 跳转至事件中心页（events.html）

#### 边界条件

| 场景 | 处理 |
|------|------|
| 无事件 | 展示空状态提示「暂无事件」，「查看全部事件」入口仍显示 |
| 标题超长 | 单行截断，省略号结尾 |
| 客户姓名缺失 | 回退使用 UID 展示 |
| 昵称未设置 | 只显示客户姓名/UID，不显示「·」 |

### §3.4 待办模块

#### 功能说明

在首页展示 BD 当前所有已加入待办且未关闭的事件摘要，按创建时间从新到旧排序。提供跳转全部待办的入口。

#### 触发方式

进入首页时自动渲染，无需手动触发。

#### 页面结构

| 元素 | 说明 |
|------|------|
| 模块标题行 | ✓ 图标 + 「待办」文字 + 数量角标（仅有待办时显示） |
| 待办事项卡片列表 | 每条一行，显示待办图标 + 事件标题 + 客户信息 + 创建时间 |
| 「查看全部待办」入口 | 固定在卡片列表底部，始终显示，点击跳转待办页 |
| 空状态提示 | 无待办时，卡片区域展示「暂无待办 / 在事件页加入待办后将出现在这里」 |

#### 字段说明（每行待办事项）

| 字段 | 说明 |
|------|------|
| ★ 图标 | 金色图标，标识该项为待办事项 |
| 事件标题 | 事件的简要描述，超长时截断（单行省略号） |
| 客户信息 | UID + 昵称（如有昵称则以「·」分隔拼接） |
| 创建时间 | 事件创建日期，格式「M月D日」，灰色展示 |
| 数量角标 | 模块标题右侧，显示当前待办总数；无待办时不显示 |

#### 排序规则

按事件创建时间降序排列（最新创建的排最前）；已关闭（closed）的事件不出现。

#### 操作结果

- 点击任意待办行 → 跳转至对应事件详情页（follow-up-thread）
- 点击「查看全部待办」→ 跳转至待办页（star.html）

#### 边界条件

| 场景 | 处理 |
|------|------|
| 无待办事项 | 展示空状态提示，「查看全部待办」入口仍显示 |
| 事件已关闭 | 不出现在列表 |
| 昵称未设置 | 只显示 UID，不显示「·」分隔符 |
| 标题超长 | 单行截断，省略号结尾 |
| 创建时间缺失 | 创建时间不展示，不占位 |

### §3.5 用户排名模块

#### 功能说明

以表格形式展示 BD 名下按「资沉」降序排列的 Top 10 客户，每行同时展示客户的本月盈亏估算及两项指标相对 滚动8周均值的变化百分比，帮助 BD 快速识别高价值客户的资产动态。

#### 触发方式

进入首页时自动渲染，数据更新时间以「更新于 M月D日 HH:MM」显示在模块标题右侧。

#### 页面结构

| 元素 | 说明 |
|------|------|
| 模块标题行 | 📊 图标 + 「用户排名」文字 + 数据更新时间（右对齐） |
| 表头行 | 客户 / 资沉（含 ? 图标）/ 7天贡献（含 ? 图标） |
| 数据行 × 10 | 每行一名客户，见字段说明 |
| 「查看全部客户」入口 | 固定在表格底部，点击跳转客户列表页 |

#### 字段说明（表头 ? 图标 Tooltip）

| 列 | Tooltip 说明文案 |
|----|-----------------|
| 资沉 | 资沉：客户过去滚动7日持仓资产均值（USDT）／变化%：过去7日资沉 vs 过去8周滚动均值 |
| 7天贡献 | 7天贡献：交易所过去滚动7日从该客户获得的盈利／变化%：过去7日贡献 vs 过去8周滚动均值 |

#### 字段说明（数据行）

| 字段 | 说明 |
|------|------|
| UID | 客户唯一标识，字重加粗 |
| 客户姓名 | UID 下方小字展示；无姓名时该行不显示 |
| 资沉数值 | K/M/B 单位格式化，金色展示；无数据时显示「—」 |
| 资沉变化% | 当前资沉 vs 滚动8周均值，正值绿色、负值红色，紧贴数值右侧；无法计算时不显示 |
| 7天贡献数值 | K/M/B 单位格式化，正值绿色、负值红色；无数据时显示「—」 |
| 7天贡献变化% | 过去7日贡献 vs 过去8周滚动均值，正值绿色、负值红色，紧贴数值右侧；无数据时不显示。**注：原型阶段用 `netContribution.wow`（周环比）作为占位，正式上线需后端提供8周均值字段** |

#### 数值格式化规则

| 绝对值范围 | 显示格式 |
|-----------|---------|
| ≥ 1,000,000,000 | X.XB |
| ≥ 1,000,000 | X.XM |
| ≥ 1,000 | X.XK |
| < 1,000 | 整数（逗号千分位） |

#### 排序规则

按资沉（7 日持仓均值）降序，取前 10 名；若资沉数据缺失，该客户排在有数据的客户之后。

#### Tooltip 交互规则

- 点击列标题旁的 ? 图标 → 在图标下方弹出说明文本框
- 点击文本框以外区域 → 关闭文本框
- 同时只能展示一个 Tooltip

#### 操作结果

- 点击任意数据行 → 跳转至对应客户详情页
- 点击「查看全部客户」→ 跳转至客户列表页（clients.html）
- 点击 ? 图标 → 展示/关闭 Tooltip 说明

#### 边界条件

| 场景 | 处理 |
|------|------|
| 客户总数 < 10 | 展示全部客户，不补位 |
| 资沉数据缺失 | 数值单元格显示「—」，变化%角标不显示 |
| 7天贡献数据缺失 | 数值单元格显示「—」，变化%角标不显示 |
| 滚动8周均值为 0 | 变化%无法计算，不显示角标 |
| 客户姓名缺失 | UID 行正常显示，姓名行不渲染（不留空行） |

### §3.6 底部 Tab 导航栏

#### 功能说明

固定在页面底部，提供首页、待办、客户、事件四个主导航入口，当前页面对应 Tab 高亮显示。

#### 触发方式

所有主页面均显示，不可折叠。

#### 页面结构

| Tab | 图标 | 标签 | 跳转目标 |
|-----|------|------|---------|
| 首页（当前高亮）| 房子图标 | 首页 | home.html |
| 待办 | 待办图标 | 待办 | star.html |
| 客户 | 用户组图标 | 客户 | clients.html |
| 事件 | 铃铛图标 | 事件 | events.html |

#### 高亮规则

当前所在页对应 Tab 图标与文字均使用金色，其余 Tab 使用灰色。

#### 边界条件

| 场景 | 处理 |
|------|------|
| 已在首页点击首页 Tab | 无操作（页面不刷新） |
| 底部安全区（iPhone 刘海屏）| Tab 栏底部预留安全区高度，避免内容被遮挡 |
