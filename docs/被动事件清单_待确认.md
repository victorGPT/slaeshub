# 被动事件清单（待确认）

## 说明

按 7 大业务模块 + 账户级别拆解所有被动事件类型。
每个事件必须有：触发条件、BD 动作对象、行动包。
标 ⚠️ 的项需要跟研发确认。

---

## 事件 vs 动态 区分原则（2026-03 圆桌结论）

### 判断标准

> **老板不干预，结果会变坏吗？**
>
> - 会变坏 → **事件（Event）**：推到首页，需要立即行动
> - 不会变坏 → **动态（Activity）**：留在客户详情页时间线，帮助了解客户

### 前提条件

只有**关注名单**（TOP 20-30 客户）内的事件才推首页。非关注名单客户的所有信息均为动态。

### 各事件分类一览

| 编号 | 事件名称 | 分类 | 推首页？ | 理由 |
|------|---------|------|---------|------|
| 1.1 | 大额提现 | **视情况** | 超大额（占资沉比例高）→ 事件；正常额度 → 动态 | 超大额可能是跑路信号，正常提现是客户自由 |
| 1.2 | 连续出金 | 动态 | 否 | 趋势性信息，不需要立即干预 |
| 1.3 | 大额入金 | 动态 | 否 | 好事，不需要紧急行动 |
| 1.4 | 资沉趋势恶化 | 动态 | 否 | 慢变量，适合在客户 Tab 关注 |
| 1.5 | 资沉趋势好转 | 动态 | 否 | 好事，不需要紧急行动 |
| 1.6 | 长期未登录 | 动态 | 否 | 客户 Tab 按最久未联系排序已覆盖 |
| 1.7 | 风控触发 | **事件** | **是** | 账户冻结不会自动好转，必须找研发 |
| 2.1 | 合约大额亏损 | 动态 | 否 | 除非极端情况 |
| 2.2 | 合约爆仓 | **事件** | **是** | 客户情绪崩溃需安抚，不干预可能流失 |
| 2.3 | 合约大额盈利 | 动态 | 否 | 好事，不需要紧急行动 |
| 3.1 | 杠杆大额亏损 | 动态 | 否 | 同合约大额亏损 |
| 3.2 | 杠杆爆仓 | **事件** | **是** | 同合约爆仓 |
| 3.3 | 杠杆大额盈利 | 动态 | 否 | 好事 |
| 4.1 | Mini 合约大额亏损 | 动态 | 否 | Mini 用户通常小额 |
| 4.2 | Mini 合约大额盈利 | 动态 | 否 | 好事 |
| 5.1 | 理财到期 | 动态 | 否 | 大部分客户自动续 |
| 5.2 | 大额理财申购 | 动态 | 否 | 好事 |
| 5.3 | 理财赎回未续存 | 动态 | 否 | 慢变量，客户 Tab 跟进 |
| 6.1 | 首次开卡 | 动态 | 否 | 好事 |

| 7.1 | 大额算力购买 | 动态 | 否 | 好事 |
| 7.2 | 算力到期 | 动态 | 否 | 不紧急 |

### MVP 首页事件（仅 3 类）

1. **风控触发**（1.7）— 过期窗口：不过期（直到解除）
2. **超大额出金**（1.1 的子集）— 过期窗口：48 小时
3. **合约/杠杆爆仓**（2.2 / 3.2）— 过期窗口：72 小时

其余所有项均为**动态**，留在客户详情页时间线。

### 1.4 资沉趋势恶化

| 项 | 内容 |
|----|------|
| 触发条件 | 8 周 AdjustedTrendRate 从稳定/增长变为流失（≤ -δ），且 R² ≥ 0.5 |
| 动作对象 | 联系客户 |
| 行动包 | 关怀话术 + 资金分布摘要 |
| 无需确认 | 触发条件由算法自动判定 |

### 1.5 资沉趋势好转

| 项 | 内容 |
|----|------|

| 行动包 | 感谢话术 + 顺势推荐 |
| 无需确认 | 触发条件由算法自动判定 |

### 1.6 长期未登录

| 项 | 内容 |

| 动作对象 | 联系客户 |

| ⚠️ 待确认 | N = 几天？14 天？30 天？ |
