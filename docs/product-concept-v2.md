# Sales Hub 产品概念 v2 — 以资金沉淀为北极星

## 1. 产品定位

让 BD（商务员）围绕客户资金沉淀，高效完成三件事：
- 看清盘子（谁最重要）
- 止损流失（谁在掉）
- 放大增长（谁在涨）

## 2. 北极星指标

```text
F = 客户在平台的总资金沉淀（折 USDT，每日 UTC 00:00 快照）
```

资金沉淀不是辅助指标，而是前台主入口。资沉收益 >> 手续费收益，所以 F 本身就是价值。

## 3. 前台结构

一个主列表 + 两个快捷筛选 Tab：

| Tab | 默认排序 | 入池条件 |
|-----|---------|---------|
| 资沉总量（默认） | mean(w1...w8) 降序 | 全量客户 |
| 资沉流失 | ChurnScore 降序 | ChurnScore > 0（公式自动过滤） |
| 资沉增长 | GrowthScore 降序 | GrowthScore > 0（公式自动过滤） |

## 4. 核心数据口径

### 4.1 资沉估值
- 统一 USDT 计价
- V_t(u) = Σ(q_i,t × p_i,t)，按持仓金额加权求和

### 4.2 时间窗
- 8 周周均值：w1, w2, ..., w8（每周日均资沉）
- w8 = 最近一周，w1 = 8 周前

### 4.3 趋势算法：线性回归 + R²

对 8 周数据拟合直线 F(t) = a + b × t：
```text
b = 斜率 = 每周资沉变化量
TrendRate = b / mean(w1...w8)    # 归一化为百分比/周
R² = 趋势可信度（0~1）
```

- TrendRate > 0：资沉在持续增长
- TrendRate < 0：资沉在持续流失
- TrendRate ≈ 0：基本稳定
- R² >= 0.7：趋势明确
- R² < 0.7：波动较大，趋势不确定

市场修正：
```text
PlatformTrendRate = 平台整体资沉的 TrendRate（同样用 8 周回归）
AdjustedTrendRate = TrendRate - PlatformTrendRate
```

### 4.4 趋势可信度

```text
Conf = 0.5 + 0.5 × R²
```

- R² = 1 → Conf = 1（趋势完全稳定）
- R² = 0 → Conf = 0.5（趋势不确定，但不会打没）

下限 0.5 是故意的：对销售来说，漏掉一个真在流失的客户，比多看一个其实没事的客户更糟。

## 5. 排序公式

### 阈值

```text
T_F = P80(mean(w1...w8))    # 前 20% 资沉客户
δ = 0.02                     # 每周偏离 2% 以上算显著
```

### 资沉总量榜

```text
TotalRank = mean(w1...w8)
```

只按盘子大小排，不掺别的因子。

### 资沉流失榜

```text
ChurnScore = log(1 + F_mean / T_F) × max(0, -AdjustedTrendRate - δ) × Conf
```

- log 压缩盘子大小：防止超大户微跌永远霸榜
- max(0, ...) 自动过滤：趋势不够负的客户 score = 0，不进入列表
- Conf 加权：趋势越稳定优先级越高

### 资沉增长榜

```text
GrowthScore = log(1 + F_mean / T_F) × max(0, AdjustedTrendRate - δ) × Conf
```

对称结构，只是方向相反。

### 入池条件

不需要单独的入池规则。公式中 `max(0, ...)` 已经自动过滤：
- AdjustedTrendRate 没超过 δ → score = 0 → 不出现在列表中
- 盘子小的客户 → log 项小 → 自然排在后面

F_mean < T_F 的客户虽然不会被硬过滤，但 log(1 + F_mean/T_F) 很小，实际排不上去。

## 6. 客户卡片（四项信息）

1. 当前资沉总量
2. 近期状态（增长 / 稳定 / 流失）
3. 原因标签
4. 建议动作

## 7. 动作约束

- 冷却：同类动作 7 天内不重复
- 互斥：同一时点只有一个主动作
- 优先级：先止损，再增长
- 留痕：每次动作记录前状态 + 事件 + 7/30 天后结果

## 8. 动作白名单

1. 销售回访
2. 续存提醒 / 权益说明
3. 资沉补贴券
4. U 卡 / 金融卡积分活动
5. 场景绑定说明

## 9. 视图冲突规则

- 流失优先于增长（先止损再增长）
- 同一客户只在一个视图中出现
- 不满足流失/增长条件的留在总量视图

## 10. MVP 不做清单

- 不做高价值 / 高潜力互斥分类
- 不做复杂客户画像体系
- 不做多业务多路径推荐
- 不做 Top-3 推荐动作
- 不做登录活跃度、可干预系数进入主排序
- 不做迁移机制
- 不做复杂 AI 归因
- 不做依赖人工标注的数据字段
