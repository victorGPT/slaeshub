# 事件模块设计文档

> 版本：v3.0 · 2026-04-01
> v3.0 变更：加入三态模型（未处理/跟进中/已处理），补充完整操作流程
> v2.0 变更：CEO Review 后收敛 MVP 范围，简化状态模型

---

## 一、概念定义

**事件（Event）** 是系统或 BD 主动发起的、需要跟进处理的客户相关任务。每条事件对应一个客户。

**设计原则：** 不区分"系统事件"和"BD 任务"——从 BD 视角看，后续动作一样（打开 → 操作 → 关闭）。"谁触发的"是属性，不是独立系统。

---

## 二、事件类型

### MVP（Phase 1）

| 事件类型 | 触发方 | BD 需要做什么 | 工具包（操作区） |
|----------|--------|-------------|-----------------|
| 大额充值 | 系统（意图引擎） | 主动联系客户，了解意图，推荐产品 | 话术模板 + 复制话术 |
| 大额提现 | 系统（意图引擎） | 挽留客户，了解提现原因 | 话术模板 + 复制话术 |
| 账户问题 | 系统 | 查询原因，协调内部处理 | 复制 UID |

**备注：**
- 触发条件由意图引擎定义，我们只接收信号并展示给 BD。
- 账户问题类话术模板后续补充。
- 礼金功能本版本不做，后续版本再加。
- 合约/杠杆爆仓不单独做事件类型，盈亏数据已在资产 Tab 展示。

### Phase 2

| 类型 | 触发方 | 说明 |
|------|--------|------|
| 主动维护 | BD 手动 | BD 主动发起，如送礼金维护关系 |
| 其他 | BD 手动 | 自定义，如客户拜访计划 |

手动创建入口：客户详情页（BD 必须先看完客户档案才能创建）。

## 三、状态流转

```
           系统触发（意图引擎 / 风控）
                     │
                     ▼
                  未处理
               /    |    \
         点"跟进"   |   点"忽略"
             │      |      │
             ▼      |      ▼
          跟进中    |    已忽略
             │      │
             ▼      ▼
           已处理 ← 直接标记

         48h 无操作（未处理状态）→ 已过期
```

### 状态定义

| 状态 | 含义 | 在哪里看得到 |
|------|------|-------------|
| 未处理 | 新事件，还没人碰 | 首页英雄卡/小卡片、事件中心"未处理"筛选 |
| 跟进中 | BD 已认领，正在处理 | 首页摘要栏"跟进中 N"、事件中心"跟进中"筛选 |
| 已处理 | 已结案 | 事件中心"已处理"筛选、客户详情时间线 |
| 已忽略 | BD 主动判断不需要处理 | 事件中心"全部"可查 |
| 已过期 | 48h 无操作，系统自动清理 | 事件中心"全部"可查 |

### 规则

- "跟进"不是门禁——操作区（话术、礼金、复制 UID）在未处理和跟进中**都可见**
- "跟进"只做一件事：**防止事件被新事件顶掉，确保 BD 能找回来**
- BD 也可以不点跟进，直接标记已处理（快速路径）
- 已处理时可选填一句结束说明（非必填）

### 过期机制

**所有事件类型统一 48h 过期。** 保证 BD 打开页面看到的都是"今天需要操作的事"，过了时效的不占位置。

| 规则 | 说明 |
|------|------|
| 过期窗口 | 触发后 48h 无操作（未处理状态） |
| 跟进中是否过期 | **不过期**——BD 已认领，说明在处理 |
| 过期后去哪 | 从首页、"未处理"列表消失，事件中心"全部"仍可查 |
| 能否恢复 | 不需要——过了时效，行动价值已大减 |

## 四、入口总览与操作路径

### 4.1 入口总览

| 入口 | 位置 | 场景 |
|------|------|------|
| A. 推送入口 | 首页英雄卡 + 小卡片 | 系统把最紧急事件推到面前 |
| B. 摘要入口 | 首页摘要栏"跟进中 N" | 找回正在处理的事件 |
| C. 任务清单 | 底部导航"事件" → 事件中心 | 主动批量消化待办队列 |
| D. 客户维度 | 客户详情页 → 事件 Tab | 查看某客户时顺带处理 |

### 4.2 操作流程

#### 流程 1：快速处理（不需要跟进的简单事件）

```
BD 打开首页
  → 看到英雄卡片：大额充值 · 张伟 · $50,000
  → 点击进入事件详情页
  → 复制话术，发给客户
  → 点"标记已处理"（可选填备注）
  → 返回首页，进度条 +1
```

**全程 1 分钟，不需要点"跟进"。**

#### 流程 2：需要时间处理（中间会被打断）

```
BD 早上 9:00 打开首页
  → 看到英雄卡片：大额提现 · 李强 · $280,000
  → 点击进入事件详情页
  → 点"跟进"（事件变为跟进中，不会被新事件顶掉）
  → 复制话术，打电话给客户
  → 客户说晚点回复，BD 先去忙别的

BD 上午 11:00 回来
  → 首页摘要栏显示：跟进中 1
  → 点击 → 跳转事件中心，自动筛选"跟进中"
  → 找到李强的事件，点进去
  → 客户已回复确认，点"标记已处理"
  → 填写备注："客户确认是购房用途，正常提现。"
  → 完成
```

**"跟进"解决的是"找回来"的问题。**

#### 流程 3：从事件中心批量处理

```
BD 打开底部导航"事件"
  → 事件中心列表
  → 筛选 pills：[ 未处理 ] [ 跟进中 ] [ 已处理 ]（默认不选 = 显示全部）
  → 统计行：共 12 个事件 · 未处理 5 · 跟进中 2
  → 逐条点开处理
```

#### 流程 4：从客户详情页处理

```
BD 在客户列表点击"李强"
  → 客户详情页（事件 Tab 默认展开）
  → 当前事件区块：显示未处理 + 跟进中的事件
  → 点击某条事件 → 进入事件详情页处理
  → 处理完返回客户详情页，事件区块刷新
```

**设计说明：**
- 第一段 = 客户详情页固定头部的完整复用（UID、资沉、变化率、备忘、标签）

- 第三段 = BD 应对工具（建议行动 + 话术参考 + 操作按钮）
- BD 看完客户信息后，结合备忘和标签，自行判断怎么沟通
- 话术模板降级为"参考"，可折叠，不是主要操作

## 八、客户详情页 · 事件 Tab 结构

### 8.1 整体布局

```
事件 Tab
│
├── 当前事件区块（行动区）
│   显示：未处理 + 跟进中的事件
│   每条：[事件类型 Tag]  状态  金额  时间  >
│   空状态："暂无待处理事件 ✓"
│
└── 跟进记录区块（浏览区）
    按时间倒序，只读
```

### 8.2 跟进记录（时间线）

**只记录两类条目：**

| 条目类型 | 写入时机 | 写入者 |
|---------|---------|--------|
| 事件触发 | 意图引擎/风控推送事件时 | 系统自动 |
| 处理结果 | BD 标记已处理时 | 系统自动（从结束说明提取） |

**不记录过程**（打了几次电话、发了什么消息）——过程记录放客户备忘（固定头部），不进时间线。

**锚点机制：** 每条跟进记录通过 `[事件类型 Tag]` 关联到对应事件。Tag 是挂钩关系的唯一标识，确保在多事件交叉时不会混淆。

**示例：**

```
跟进记录
│
● 04-01 14:32  张三
  [大额提现] 已处理 — "客户已确认，正常操作。"
│
● 04-01 09:00  系统
  [大额提现] 触发 — $280,000
│
● 03-31 16:00  张三
  [账户异常] 已处理 — "已联系研发解封。"
│
● 03-30 11:00  系统
  [账户异常] 触发 — 风控封禁
│
● 03-28 10:00  系统
  [大额充值] 触发 — $50,000
```

**数据流：** BD 在事件详情页点"标记已处理" → 填写结束说明 → 系统自动在该客户的跟进记录中追加一条 `[事件类型] 已处理 — "结束说明内容"`。BD 不需要手动操作时间线。

### 8.3 设计原则

- 事件类型以 Tag 形式显示，不是卡片
- 时间线纯浏览，不承载操作
- 跟进过程记录放客户备忘，不进时间线
- 时间线只记录：事件触发 + 处理结果
- Tag 是事件和处理结果的逻辑锚点

## 十、Phase 2 Roadmap

| 功能 | 前置条件 | 验证目标 |
|------|----------|----------|
| "跟进"门禁 + 操作区解锁 | MVP 数据表明 BD 直接标记处理过于草率 | 门禁能否提升处理质量 |
| 必填结束说明 | 门禁验证通过 | 结束说明是否有复盘价值 |
| 自动忽略 + 恢复 | 后端支持 | 减少事件堆积 |
| 手动创建事件 | MVP 闭环验证 | BD 是否需要自建任务 |
