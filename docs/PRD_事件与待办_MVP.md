## §1 术语定义

| 术语 | 定义 |
|------|------|
| **事件** | BD 需要处理的工作单元，每条对应一个客户，分系统事件和 BD 事件两种来源 |
| **系统事件** | 系统根据客户行为自动生成（大额充值、大额提现、大额盈利、大额亏损）|
| **BD 事件** | BD 手动创建（主动联系、客户反馈、其他）|
| **加入待办** | BD 对事件打星标，标记后事件出现在待办页；不改变事件处理流程 |
| **待办** | 已加入待办（星标）且未关闭的事件集合 |
| **行动工具包** | 事件详情页内嵌的决策辅助模块，提供情境分析、推荐策略和即用话术 |
| **BD** | 使用本系统的业务拓展人员 |

## §2 模块概述

### §2.1 定位

事件页是 BD 跨所有客户的统一工作 inbox，将系统自动触发和 BD 手动创建的事件汇聚在一处。

### §2.2 MVP 范围

**做**
- 系统事件：大额充值、大额提现、大额盈利、大额亏损
- BD 手动创建事件：主动联系、客户反馈、其他
- 事件页（全量列表 + 类型筛选）
- 待办页（星标事件列表 + 快速创建）
- 事件详情页（BD 事件 / 系统事件各一个详情页）
- 行动工具包（话术跟进、发放礼金、面谈邀约）
- 客户详情事件 Tab（该客户的事件卡片）
- 客户详情 Header 活跃跟进快捷行
- 首页待办模块 + 事件跑马灯
- 客户列表跟进状态圆点

**不做**
- 意图引擎（算法推送）— MVP 阶段系统事件由 mock 数据模拟；正式上线时需定义以下触发阈值（部分已有参考值，**仍需产品 + 数据团队最终确认**）：
  - 大额充值：单笔充值金额 ≥ 总资产 30%（参考 docs/话术模板库.md）
  - 大额提现：单笔或近期累计提现 ≥ 总资产 20%（参考 docs/话术模板库.md）
  - 大额盈利：阈值待定
  - 大额亏损：阈值待定
- 逾期 Push 通知
- 截止日期手动设置（加入待办时统一为「待定」）

## §3 功能详述

> 阅读约定：每个功能模块按以下顺序描述——**功能说明 → 触发方式 → 页面/弹窗结构 → 字段说明 → 操作结果 → 错误/空状态**。

### §3.1 事件数据来源

事件分两种来源，共用同一份数据（`follow_up_events`）。

| 来源 | 类型 | 产生方式 |
|------|------|---------|
| 系统事件 | 大额充值 / 大额提现 / 大额盈利 / 大额亏损 / 账户封禁 | 系统自动生成，MVP 阶段由 mock 数据模拟 |
| BD 事件 | 主动联系 / 客户反馈 / 其他 | BD 在待办页手动创建 |

系统事件标题示例：「充值 $126万，占资产 30%」、「7日亏损 $58万，占资产 5%」。  
BD 事件标题规则：`类型 + (说明 ? '：' + 说明 : '')`，例：「主动联系：跟进合约意向」。

### §3.2 事件页

**功能说明**：BD 查看所有事件的汇总列表，可按类型筛选，点击进入详情。

**触发方式**：点击底部导航「事件」。

**页面结构**

| 区域 | 内容 |
|------|------|
| 顶部标题 | 「事件」|
| 筛选 pills | 全部 / 主动联系 / 客户反馈 / 大额充值 / 大额提现 / 大额盈利 / 大额亏损 / 账户封禁 / 其他 |
| 事件卡片列表 | 按创建时间倒序；已关闭事件置灰 + 标题加删除线 |

**事件卡片字段**：类型 badge + 事件标题 + 客户 UID · 昵称 + 创建时间

**操作**

| 操作 | 前置条件 | 结果 |
|------|---------|------|
| 点击筛选 pill | 无 | 列表按该类型过滤；点「全部」恢复全量 |
| 点击事件卡片 | 无 | BD 事件 → BD 详情页；系统事件 → 系统详情页 |

**错误/空状态**

| 触发条件 | 显示 |
|---------|------|
| 无任何事件 | 「暂无事件」|
| 当前筛选类型下无事件 | 「暂无事件」|

### §3.3 待办页

**功能说明**：BD 查看自己星标（加入待办）且未关闭的事件，并可手动创建新事件。

**触发方式**：点击底部导航「待办」。

**页面结构**

| 区域 | 内容 |
|------|------|
| 顶部标题 | 「待办」|
| 事件卡片列表 | 仅显示星标且未关闭的事件；卡片格式与事件页一致 |
| FAB 按钮 | 右下角金色「+」|

**排序规则**：按创建时间降序排列（最新创建的排最前）。

**操作**

| 操作 | 前置条件 | 结果 |
|------|---------|------|
| 点击事件卡片 | 无 | 跳转对应详情页（规则同事件页）|
| 点击「+」FAB | 无 | 打开「添加事件」Sheet（见 §3.3.1）|

**错误/空状态**

| 触发条件 | 显示 |
|---------|------|
| 无星标事件 | 「暂无特别关注事件」|

#### §3.3.1 添加事件（Sheet）

**功能说明**：BD 手动创建一条跟进事件，创建后自动加入待办。

**触发方式**：点击待办页右下角「+」FAB。

**弹窗结构**：从底部弹出的 Sheet，包含标题栏、客户搜索区、类型选择区、说明输入区、提交按钮。

**字段说明**

| 字段 | 类型 | 是否必填 | 说明 |
|------|------|---------|------|
| 客户 | 搜索单选 | 必填 | 按 UID 或昵称实时搜索；已选后显示 UID + 昵称 + 清除按钮 |
| 事件类型 | 单选 pill | 必填 | 主动联系 / 客户反馈 / 其他，三选一 |
| 事件说明 | 多行文本 | 选填 | 留空则标题仅为类型名 |

**操作结果**：点「添加」后——
- 事件写入列表顶部
- 自动加入待办
- Sheet 关闭，待办列表刷新

同一客户允许同时存在多条事件，互不影响。

**错误/空状态**

| 触发条件 | 显示 |
|---------|------|
| 客户或类型未填 | 「添加」按钮禁用（灰色不可点击）|
| 搜索无匹配客户 | 「无匹配客户」|

### §3.4 事件详情页

**功能说明**：查看事件详情（系统事件和 BD 事件共用同一详情页），并执行加入待办或标记完成操作。

**触发方式**：点击任意列表中的事件卡片（事件页 / 待办页 / 首页 / 客户详情事件 Tab）。

**页面结构**

| 区域 | 内容 |
|------|------|
| 顶部导航 | 返回按钮 + 「事件」标题 |
| 事件卡 | 类型 badge + 来源 badge（BD 或 系统）+ 创建时间 + 标题 + 说明（BD 事件有则显示）|
| 客户行 | 客户 UID + 昵称 + 「查看详情 →」|
| 行动工具包 | 见 §3.6 |
| 底部操作按钮 | 单按钮，文案随状态切换 |

**操作**

| 操作 | 前置条件 | 结果 |
|------|---------|------|
| 点击「加入待办」| 事件未在待办中 | 加入待办，返回上一页 |
| 点击「已完成」| 事件已在待办中 | 从待办移除 + 标记已关闭，返回上一页 |

**底部按钮样式**

| 状态 | 按钮文案 | 样式 |
|------|---------|------|
| 未加入待办 | 「加入待办」| 金色 |
| 已加入待办 | 「已完成」| 灰色 |

**错误/空状态**

| 触发条件 | 显示 |
|---------|------|
| fuId 对应事件不存在 | 「未找到跟进事件」|

### §3.6 行动工具包

**功能说明**：内嵌在两种事件详情页中，根据事件类型和客户资沉排名，自动推荐跟进策略并提供即用话术。

**触发方式**：进入事件详情页时自动加载，无需手动触发。

**面板结构**

| 区域 | 内容 |
|------|------|
| 情境分析 | 系统自动生成的一段文字，描述当前客户形势（见下表）|
| 推荐策略标签 | 显示当前推荐策略名称 |
| 策略面板 | 根据推荐策略展示对应面板（话术跟进 / 发放礼金 / 面谈邀约）|
| 其他策略 | 折叠区，展开后可切换到其余两种策略 |

**话术跟进面板**：5 种风格 pill + 话术内容 + 「复制话术」按钮

**发放礼金面板**：系统建议金额 + 建议理由 + 5 种风格 pill + 参考话术 + 「复制话术」按钮

**面谈邀约面板**：5 种风格 pill + 邀约话术 + 「复制邀约话术」按钮

**话术风格**（5 种）：简洁直接 / 亲切随和 / 专业正式 / 关怀体贴 / 积极引导

> 各场景话术原文见 **[docs/话术模板库.md](./话术模板库.md)**，覆盖大额提现、大额充值、账户风控、爆仓、大额亏损、大额盈利共 6 个场景，每场景 5 种风格。PRD 不重复维护话术内容，以话术模板库为准。
>
> ⚠️ **当前原型中话术内容为 mock 硬编码**，研发上线时需按话术模板库实际内容对接，不可直接沿用原型 JS 中的字符串。

**推荐策略规则**（按优先级判断，命中第一条即止）

| 优先级 | 事件类型 | 客户条件 | 推荐策略 |
|--------|---------|---------|---------|
| 1 | 大额充值 / 大额盈利 | 资沉排名 ≤ 20 或利润排名 ≤ 20 | 发放礼金 |
| 2 | 大额充值 / 大额盈利 | 资沉排名 ≤ 50 | 面谈邀约 |
| 3 | 其余所有类型 | — | 话术跟进（默认）|

> 资沉排名：按客户 fMean 在全客户中的升序排名；利润排名：按 netContribution.total 排名。

**礼金建议金额**

| 资沉排名 | 建议金额 |
|---------|---------|
| 前 10 | 1000U |
| 前 11–20 | 500U |
| 其余 | 200U |

**情境分析文案**（按事件类型自动匹配）

| 事件类型 | 文案 |
|---------|------|
| 大额充值 | 客户发生大额充值，{资沉描述}，建议主动跟进，巩固资金留存意愿 |
| 大额提现 | 客户发生大额提现，存在资金流失风险，建议立即联系了解原因并尝试挽留 |
| 大额盈利 | 客户近期账户大幅盈利，情绪良好，是深化关系、发展长期信任的好时机 |
| 大额亏损 | 客户近期出现较大亏损，情绪可能较低落，建议主动关怀，切忌施压（含爆仓场景，话术见模板库场景四）|
| 账户封禁 | 客户账户遭到封禁，影响正常操作，需紧急跟进协助处理，避免客户流失 |
| 主动联系 | 您主动发起了对该客户的联系，建议结合客户近期动态选择合适的话题切入 |
| 客户反馈 | 客户有主动反馈，说明参与度较高，建议及时回应并转化为深度关系 |
| 其他 | 建议根据客户当前情况，选择合适的跟进策略主动维系关系 |

> {资沉描述}：排名 ≤ 20 → 「资沉排名前20，属核心高价值客户」；≤ 50 → 「资沉排名前50，属优质客户」；其余 → 「属活跃客户」

**操作**

| 操作 | 前置条件 | 结果 |
|------|---------|------|
| 切换话术风格 pill | 无 | 话术内容随之更新 |
| 点击「复制话术」| 无 | 话术写入剪贴板；按钮变「已复制」，1.5 秒后恢复 |
| 点击「其他策略」| 无 | 折叠区展开，显示另外两种策略切换按钮 |
| 切换到其他策略 | 无 | 主面板切换，折叠区收起 |

#### §3.8.2 事件跑马灯模块

**功能说明**：在首页实时展示最新事件动态，条数多时自动滚动。

**触发方式**：进入首页时自动渲染。

**模块结构**

| 区域 | 内容 |
|------|------|
| 模块标题 | 铃铛图标 + 「事件」+ 灰色事件总数 badge |
| 每条展示 | 类型彩色 badge + 事件标题 + 客户 UID · 昵称 |

**滚动规则**：≤ 3 条静态展示；> 3 条自动垂直滚动。

**操作**

| 操作 | 前置条件 | 结果 |

## 附录 A：事件状态转移

```
[待办页 FAB 创建 BD 事件]
      │ 自动加入 slaesh_flagged（deadline: pending）
      ↓
  出现在待办页 + 事件，详情页底部：「已完成」
      │ 点「已完成」
      ↓
  从 slaesh_flagged 移除 + 加入 slaesh_closed，卡片置灰

[事件未星标事件（含系统事件）]
  详情页底部：「加入待办」
      │ 点「加入待办」
      ↓
  加入 slaesh_flagged（deadline: pending）→ 流程同上
```
