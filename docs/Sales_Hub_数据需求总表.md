## 核心算法说明

### 1. 综合净贡献

平台从该客户身上赚了多少净利润。

```
净利润 = 双仓合约 + 超级杠杆 + Mini合约 + 金融卡 + 云算力 - 理财利息 - 返佣
```

**5 个收入模块**：
- 双仓合约：手续费 + 非对冲订单的用户亏损
- 超级杠杆：手续费 + 非对冲订单的用户亏损
- Mini合约：手续费 + 资金费率 - 用户盈亏
- 金融卡：开卡费 + 消费手续费
- 云算力：算力费用 + 电费

**2 个成本模块**：
- 理财利息：平台付给客户的利息（纯支出）
- 返佣：平台付给渠道代理的佣金（纯支出）

### 2. 事件触发条件

| 事件类型 | 触发条件 | 过期窗口 |
|---------|---------|---------|
| 风控触发 | 风控系统标记账户异常（冻结/限制/KYC） | 不过期 |
| 超大额出金 | 单笔出金 ≥ 客户资沉的 30% | 48 小时 |
| 爆仓 | 合约或杠杆仓位被强制平仓 | 72 小时 |

### 3. 资沉流动的流入/流出

- **流入** = 客户从外部向平台充值的金额
- **流出** = 客户从平台向外部提现的金额
- 与资沉变化不同：资沉还受交易盈亏、币价波动等影响，流入/流出只计算真实的资金进出

## 数据来源系统

| 系统 | 提供什么 | 用在哪 |
|------|---------|-------|
| 用户系统 | 客户 UID | 头部身份 |
| 资产系统 | 周度资沉快照、各子账户余额 | 头部资沉、资金分布 |
| 充提系统 | 每周充值/提现汇总 | 资沉流动图表、超大额出金事件 |
| 交易系统 | 各业务线交易数据、用户盈亏 | 净贡献、客户盈亏、爆仓事件 |
| 风控系统 | 风控触发记录 | 事件系统 |
| 理财系统 | 已发放利息 | 净贡献（成本项） |
| 返佣系统 | 已结算返佣 | 净贡献（成本项） |
| 福利金系统 | 剩余额度 | 事件线程（发放福利金） |
| 跟进系统 | 跟进记录 | 事件 Tab 跟进时间线 |

## MVP 模拟 → 正式产品需要的接口

| 模拟内容 | MVP 怎么做的 | 正式产品需要什么 |
|---------|-----------|---------------|
| 资沉数据 | 前端随机生成 9 周序列 | 资产系统：周度资沉快照接口 |
| 资金分布 | 前端按比例随机分配 | 资产系统：各子账户余额查询 |
| 流入/流出 | 前端从资沉差值推算 | 充提系统：周度充提汇总接口 |
| 事件 | 前端按概率生成 | 事件系统：事件列表接口（按客户/状态筛选） |
| 事件处理状态 | sessionStorage（关页即丢） | 事件系统：状态更新接口 |
| 净贡献 | 前端随机生成 | 用户贡献收益查询系统接口 |
| 客户盈亏 | 前端随机生成 | 交易系统：用户盈亏汇总接口 |
| 跟进记录 | 硬编码 3 条示例 | 跟进系统：CRUD 接口 |
| 收藏/备忘/标签 | localStorage | 后端持久化接口 |
